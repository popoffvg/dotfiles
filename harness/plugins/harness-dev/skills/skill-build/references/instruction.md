# Skill shape: instruction (rules & reference)

Author a skill whose body is a **flat set of rules, definitions, or facts** the agent consults on demand — a review checklist, a style guide, a glossary. No ordered steps; order does not matter, so a flat peer-set is correct here, not a smell.

Shared principles (pruning, leading words, failure modes — incl. positive-not-negation phrasing): `references/foundations.md`.

## Structure

- Each rule stands alone under its own heading or bullet — every rule on one rung.
- **Co-locate**: keep a concept's definition, rules, and caveats under one heading, so reading one part brings its neighbours.

## Make each rule bind

- Tie the set to an **exhaustive** gate — "every rule applied" — so flat reference drives thorough legwork the way ordered steps do.
- Phrase each rule as the **positive** target behaviour (see foundations → Negation).

## Output

`SKILL.md` with frontmatter + the flat rule-set, co-located by concept, each rule positive and load-bearing.
