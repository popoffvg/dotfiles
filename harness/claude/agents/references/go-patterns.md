# Go Patterns — Non-Obvious Reference

Patterns the model frequently gets wrong, skips, or implements in outdated style.

---

## Function Signatures — Params Struct Pattern

Models default to functional options (`WithX` closures) or long parameter lists. Prefer: required params explicit, optional params as a `FuncNameParams` struct.

```go
type NewServerParams struct {
    Timeout time.Duration // zero value = use default
    Logger  *slog.Logger  // nil = use default
    TLS     *tls.Config   // nil = no TLS
}

func NewServer(addr string, params NewServerParams) *Server {
    return &Server{
        addr:    addr,
        timeout: cmp.Or(params.Timeout, 30*time.Second),
        logger:  cmp.Or(params.Logger, slog.Default()),
        tls:     params.TLS,
    }
}

// Usage:
srv := NewServer(":8080", NewServerParams{Timeout: 10 * time.Second})

// All defaults:
srv := NewServer(":8080", NewServerParams{})
```

**Rules:**
- Required params = explicit function arguments (compiler enforces)
- Optional params = `FuncNameParams` struct (zero values = defaults)
- Use `cmp.Or` (Go 1.22+) to apply defaults for zero values
- Struct name follows `FuncNameParams` convention (not `Options`, not `Config`)
- Use functional options (`WithX`) ONLY for public library APIs where backwards compatibility matters

## errgroup with Context Cancellation

Models often use `sync.WaitGroup` when `errgroup` is correct, or forget that `errgroup.WithContext` cancels on first error.

```go
func fetchAll(ctx context.Context, urls []string) ([]Response, error) {
    g, ctx := errgroup.WithContext(ctx)
    results := make([]Response, len(urls))

    for i, url := range urls {
        g.Go(func() error {
            resp, err := fetch(ctx, url)
            if err != nil {
                return fmt.Errorf("fetch %s: %w", url, err)
            }
            results[i] = resp // safe: unique index per goroutine
            return nil
        })
    }
    return results, g.Wait()
}
```

**Rules:**
- Use `errgroup.WithContext` when any failure should cancel siblings
- Use plain `errgroup.Group` when all goroutines must complete regardless
- Each goroutine writes to its own index — no mutex needed
- For bounded concurrency: `g.SetLimit(n)`

## Golden Files Testing

Rarely generated by models. Essential for CLI tools, formatters, serializers.

```go
var update = flag.Bool("update", false, "update golden files")

func TestRender(t *testing.T) {
    got := render(input)
    golden := filepath.Join("testdata", t.Name()+".golden")

    if *update {
        os.WriteFile(golden, []byte(got), 0o644)
        return
    }

    want, err := os.ReadFile(golden)
    if err != nil {
        t.Fatal(err)
    }
    if diff := cmp.Diff(string(want), got); diff != "" {
        t.Errorf("mismatch (-want +got):\n%s", diff)
    }
}
```

Run `go test -update` to regenerate golden files after intentional changes.

## Testcontainers for Integration Tests

Models default to mocks when real containers are better. Use `testcontainers-go` for DB/queue integration tests.

```go
func TestWithPostgres(t *testing.T) {
    if testing.Short() {
        t.Skip("skipping integration test")
    }
    ctx := t.Context()
    pg, err := postgres.Run(ctx,
        "postgres:16-alpine",
        postgres.WithDatabase("testdb"),
        postgres.WithUsername("test"),
        postgres.WithPassword("test"),
        testcontainers.WithWaitStrategy(
            wait.ForLog("database system is ready").
                WithOccurrence(2).WithStartupTimeout(5*time.Second),
        ),
    )
    testcontainers.CleanupContainer(t, pg)
    require.NoError(t, err)

    connStr, err := pg.ConnectionString(ctx, "sslmode=disable")
    require.NoError(t, err)
    // test against real Postgres
}
```

## sync.Pool (correct usage)

Models frequently misuse: forgetting Reset, storing pointers to pool objects, or using Pool for small values where stack allocation is cheaper.

```go
var bufPool = sync.Pool{
    New: func() any { return new(bytes.Buffer) },
}

func process(data []byte) string {
    buf := bufPool.Get().(*bytes.Buffer)
    defer func() {
        buf.Reset()  // MUST reset before returning
        bufPool.Put(buf)
    }()
    buf.Write(data)
    return buf.String()
}
```

**Rules:**
- ALWAYS `Reset()` before `Put()` — stale data leaks otherwise
- Never store references to pooled objects beyond the Get/Put scope
- Only worth it on hot paths — profile first with `go test -benchmem`
- Objects may be collected at any GC — Pool is a cache, not permanent storage

## Escape Analysis Awareness

Models ignore allocation cost. On hot paths, these matter:

```go
// BAD: forces heap allocation (returns interface)
func compute(x int) any { return x }

// GOOD: concrete return stays on stack
func compute(x int) int { return x }

// BAD: pointer escapes to heap
func newItem() *Item { return &Item{} }

// GOOD: let caller decide (value on stack, pointer if needed)
func newItem() Item { return Item{} }

// GOOD: append to caller's slice — no escape
func appendFiltered(dst, src []Item, pred func(Item) bool) []Item {
    for _, item := range src {
        if pred(item) {
            dst = append(dst, item)
        }
    }
    return dst
}
```

**Diagnostic:** `go build -gcflags='-m' ./...` shows escape decisions.

## Pre-allocation

Models always write `var result []T` + append. On known-size collections, pre-allocate.

```go
// BAD: grows from 0
var results []Result
for _, item := range items {
    results = append(results, transform(item))
}

// GOOD: single allocation
results := make([]Result, 0, len(items))
for _, item := range items {
    results = append(results, transform(item))
}

// GOOD: reusable buffer pattern
type Encoder struct {
    buf []byte
}

func (e *Encoder) Encode(v any) []byte {
    e.buf = e.buf[:0] // reset length, keep capacity
    e.buf = appendJSON(e.buf, v)
    return e.buf
}
```

## Channel Close Semantics

Models often close channels unnecessarily or from the wrong goroutine.

**Rules:**
- Channels are garbage-collected — `close()` is for **signaling**, not cleanup
- Only the **sender** closes — never the receiver
- If no receiver needs the "done" signal, don't close at all
- Closing a closed channel panics — use `sync.Once` if multiple senders

```go
// GOOD: sender closes to signal "no more items"
func generate(ctx context.Context, items []int) <-chan int {
    ch := make(chan int)
    go func() {
        defer close(ch) // sender closes
        for _, item := range items {
            select {
            case ch <- item:
            case <-ctx.Done():
                return
            }
        }
    }()
    return ch
}

// BAD: closing from receiver side, or closing when nobody ranges over it
```

## Interface at Consumer, Not Producer

Models define interfaces next to the implementation. Go idiom: define where consumed.

```go
// BAD: interface in the "email" package next to implementation
package email
type Sender interface { Send(ctx context.Context, to, body string) error }
type Client struct{}
func (c *Client) Send(...) error { ... }

// GOOD: interface in the consumer package
package notifier
type Sender interface { Send(ctx context.Context, to, body string) error }
type Service struct { sender Sender }

// email.Client satisfies notifier.Sender implicitly — no import needed
```
